# AI 时代的 Cognitive Loop

最近读到 Microsoft CEO Satya Nadella 那篇 token capital 长文（40k+ likes、65M+ views），一句话扎心：

> 你可以将任务甚至整个工作外包出去，**但你永远无法摆脱学习的责任**。

对个人来说，这件事变成一个具体问题：**AI 写得越快越像样，你的判断力反而越值钱**。

## 三个反常识

**1. 思考可以外包，理解不能**

AI 每段产出后，强制问自己「这段我能拿出去解释吗」。不能解释的不用——你没经过这段判断，相当于没看。

**2. Build the loop is easy，knowing what's worth encoding is the hard part**

把流程做成 skill / CLAUDE.md / 自动化脚本——技术上不难。难的是**判断哪些值得 encode**：

- ✅ 高频（一周 3+ 次）
- ✅ 高判断（涉及取舍 / 品味）
- ✅ 重复犯错（同坑栽 2+ 次）

一次性的事情不 encode；没反复出现的 pattern 不 encode（会形成 false 规则）。

**3. Own the loop，不 own the model**

今天 Claude，明天 Gemini，后天某个开源 model —— 模型是 commodity
你拥有的应该是：prompt / skill / eval / trace / judgment

切换模型时，你的资产应该 90%+ reusable。如果不是，说明你 lock-in 了。

## 三个测试

判断你是否真有 cognitive loop：

| 测试 | 通过标准 |
|------|---------|
| 解释测试 | 能用一句话解释为什么 AI 这次输出对/错 |
| 发布测试 | 你的判断能写成 X / 公众号让别人看到 |
| 迁移测试 | 切换到另一个 AI 工具，资产能带走 90%+ |

三个测试都过 = 你有 cognitive loop。否则只是 AI 工具使用记录。
