# X (Twitter) 线程切分规则

> 生成 X 线程时读这个文件。

## 一、字符计数规则（不要靠 text.length）

X 的字符权重不是 1 字 = 1 字符，规则如下：

| 字符类型 | 权重 |
|----------|------|
| 英文字母 / 数字 / 标点 | 1 |
| 中文 / 日文 / 韩文 | 2 |
| Emoji | 2 |
| URL（任何长度） | 23 |

**单条上限**：280 加权字符。

**校验工具**（Phase 3 写代码时用）：[`twitter-text`](https://github.com/twitter/twitter-text) 官方库的 `parseTweet(text).weightedLength`。

**Phase 2（在 Claude Code 跑）时**，你按下面这个手算公式估：
- 纯中文段：`字数 × 2`
- 中英混排：`英文字符 + 中文字符 × 2`
- 有 URL：减去 URL 长度，加 23

## 二、线程切分原则

### 1. 总条数

- **3-12 条**最佳
- **1-2 条**：内容太薄，不如不发线程
- **超过 12 条**：拆成两个独立线程，或者改发公众号

### 2. 第 1 条：Hook（钩子）

**必须满足**：
- 一句话讲清楚整个线程的核心冲突 / 反差 / 数字
- 留悬念，不剧透结论
- 读者读完第 1 条会想"为什么？" / "怎么做的？"

**不要的**：
- 不写 "🧵 1/N"，除非原作者风格如此
- 不开场白 "I have something to share..."
- 不写 "thread 🧵"，看起来像 thread bot

### 3. 第 2 条到倒数第 2 条：内容

- 每条**自成一意**，读者只读这一条也能理解
- 但**相邻两条之间有钩子**："see why ↓" / "but here's the catch:" / "更狠的是："
- **段落边界切**，绝对不切句子中间
- **代码 / 数据**单独一条，不混排
- **截图**：在该条结尾写 `[Image: 描述]` 占位，提示用户附图

### 4. 最后一条：CTA / Takeaway

- 总结核心 + 一个具体行动 / 提问
- **不要硬卖**：「关注我学更多」是 X 上最低级的 CTA
- 优雅的 CTA：转推、回复、点开 profile 看 pinned tweet

## 三、tag / mention 规则

- **正文里不放 #tag**：X 算法对 hashtag 比较冷淡，会显得像 SEO 党
- **结尾可以放 1-2 个相关 tag**：`#AI` `#Claude` 类
- **mention 别人**：只在有真实关联时（"原作者 @xxx"）；不刷流量

## 四、长文 → 线程转换示例

**原文段落**（公众号风格，假设 500 字）：
> 很多人问我怎么涨粉。我发现一个反常识的事实：涨粉最快的人，不是最努力的人。他们的共同点是——他们都在用一种"不发朋友圈"的方式做内容。这听起来反直觉，但数据上看，过去 30 天我用这套方法涨了 5 万粉。

**X 线程化**（5 条）：
> **1**：涨粉最快的人，不是最努力的人。他们的共同点是一种"不发朋友圈"的方式。30 天涨 5 万粉的实际验证 ↓
>
> **2**：先说"努力"为什么没用。每天发 3 条朋友圈、每天写 1 篇笔记、每天回 100 条评论——这都是"高频低质"的努力。算法不奖励高频，奖励高完播率。
>
> **3**：我的方法是反过来：每周发 1 条，但每条都是过去 7 天最重要的认知。不发流水账，不发"今天看了一本书"。发"读完这本书我改了一个习惯"。
>
> **4**：数据：发布频率从每天 3 条降到每周 1 条，单条平均播放从 800 → 12000，30 天新增 5.2 万粉。降频反而涨粉。
>
> **5**：核心：算法奖励"完播 + 互动"，不奖励数量。低频高质内容，单条带来的曝光等于过去 20 条总和。
> 
> 行动：本周只发 1 条，但把它做到你能力上限。

## 五、Gotchas 清单

- **不要每条都加 emoji 当 bullet**：`🔥 / 💡 / 🚀` 堆砌是 AI 味
- **不要每条结尾问"你觉得呢？"**：互动感是假的，读者会跳过
- **代码块**：X 不支持代码格式，纯文本贴。如果代码很长，建议改"截图发"
- **链接**：放在最后一条，不要中间打断
- **中英混排**：注意权重计算（中文 ×2），手算估计时往大里估，留 10 字符 buffer
- **第一条权重最容易爆**：因为要塞核心信息。如果第一条 > 280，拆成"hook + 立刻补充"两条，第一条精简到 < 200

## 六、输出格式

```
Post 1 (hook):
> {内容}
> (估算 ~{N}/280)

Post 2:
> {内容}
> (估算 ~{N}/280)

...

Tag: #tag1 #tag2

注：实际发布前用 twitter-text 库精确校验，估算可能差 ±10 字符
```

## 七、输出前自检

- [ ] 第 1 条是 hook，不是介绍
- [ ] 没有 "1/N" 标号（除非原作者风格）
- [ ] 每条独立成意，又有连贯钩子
- [ ] 最后一条是 takeaway 或具体行动
- [ ] 中文权重 ×2 估算过
- [ ] 没有"thread 🧵" 开场
- [ ] 没有 "你觉得呢？" 假互动
