# 小红书图文格式规则

> 生成小红书内容时读这个文件 + `prompts/dbs-xhs-titles.md`（75 公式）。

## 一、小红书内容结构

一篇标准小红书图文 =
- **标题**（≤ 20 字含标点，硬上限）
- **封面图**（1 张，竖版 3:4，标题大字 + 副标题）
- **内容图**（2-8 张，竖版 3:4 或方形 1:1，每张承载 1 个核心观点）
- **正文**（200-500 字，发布框里）
- **tag**（5-10 个，结尾）

## 二、标题（强制读 prompts/dbs-xhs-titles.md）

### 硬规则

1. **≤ 20 字（含标点）**，硬上限
2. **必须从 75 公式里挑**，每个标题标注公式编号
3. **至少给 3 个候选**，覆盖不同触发器类型
4. **Top 1 推荐**带一句话理由

### 5 条铁律（来自 dbs-xhs-title）

1. **留悬念，不剧透**：用「为什么」不用「证明」，看完标题观众还需要点进来
2. **话题扩大不缩窄**：用「赚钱」不用「知识付费」、用「不上班」不用「自媒体」
3. **击中真实痛点**：「一人公司」是表象，「不上班」才是真实痛点
4. **张力检查命中 2 项**：对比/反差、具体数字、悬念/好奇、冲突/争议、时间承诺、结果承诺 —— 6 项命中 ≥ 2
5. **≤ 20 字**：标点算字数

## 三、正文（发布框内）

### 结构

```
{1-2 句开头钩子，对应封面标题的悬念}

{正文 3-5 段，每段 1-3 句，emoji 适度}

{结尾互动引导，1 句}
```

### 写作规则

- **emoji 用法**：可以用，但每段最多 1-2 个，绝对不堆叠
- **段落短**：每段 1-3 句，移动端阅读
- **不要"姐妹们"开场**：dontbesilent 文字洁癖原则，但如果原文行业本身就用（比如美妆），可酌情保留
- **不要"码住"等土味互动话术**：太过时
- **结尾引导**：用问句或挑战，不用"点赞收藏关注三连"

### 互动引导例子

✅ 好：「你试过这个方法吗？评论区告诉我效果。」
✅ 好：「如果你也犯过第 3 个错，扣 1。」
❌ 烂：「点赞收藏关注三连，更新更多干货！」
❌ 烂：「家人们觉得有用吗？」

## 四、卡片图分段（关键）

**封面图**（1 张）：
- 大字标题（≤12 字，比正文标题更精炼）
- 小字副标题/钩子
- 竖版 3:4，适合双指放大查看
- 配色：暖色（米黄/暖白）或瑞士风（黑白红）

**内容图**（2-8 张）：
- 每张 1 个核心观点（不要塞 3 个）
- 大字主点 + 小字解释
- 数字/反差/金句优先
- 序号「02 / 03 / 04」帮助引导滑动

**末卡**（1 张）：
- 总结 + 一句行动建议
- 不要做"关注我"，做"今天就试这一招"

### 输出格式

```
**封面**：
- 主标题：{≤12字}
- 副标题：{钩子，1 句}

**卡 2 - "{主题}"**：
- 主点：{核心观点}
- 解释：{1-2 句}

**卡 3 - "{主题}"**：
- 主点：{核心观点}
- 解释：{1-2 句}

...

**末卡 - "{总结主题}"**：
- 主点：{行动建议}
- 解释：{为什么}
```

> Phase 3 由代码渲染成 PNG，Phase 2 你只给文字结构即可

## 五、tag 规则

### 选 tag 公式

1. **2-3 个大词**（高流量但竞争激烈）：`#职场` `#自媒体` `#个人成长`
2. **2-3 个中词**（精准匹配）：`#知识博主` `#小红书运营` `#副业避坑`
3. **1-2 个长尾**（精准目标人群）：`#30岁创业` `#内向博主`

### 数量

- **5-10 个**最佳
- 少于 5：流量池打不开
- 多于 10：算法判定为凑数

### Gotchas

- **不要塞热门但无关的 tag**：比如内容是 AI 但塞 `#穿搭` —— 会被限流
- **不要英文 tag**：除非内容本身是英文领域
- **#tag 之间空一格**：`#职场 #自媒体`，不要连写

## 六、内容形式诊断

不是所有长文都适合做小红书图文：

| 原文特性 | 是否适合 |
|----------|---------|
| 反常识观点 + 1-2 个数据 | ✅ 完美 |
| 流程教程 | ✅ 适合，每步一张卡 |
| 长论证逻辑链 | ⚠️ 拆不开，建议简化成 1 张观点卡 + 引导看公众号 |
| 纯流水账 / 日记 | ❌ 不适合，发不成 |
| 大量代码 | ❌ 不适合 |
| 时事评论 | ⚠️ 小红书算法对时事冷淡 |

不适合的，**告诉用户**，不要硬出。

## 七、Gotchas 清单

- **不要硬凑 6 张图**：原文撑不起就 3 张，质量优先
- **数字别编**：原文没说"30 天涨 5 万粉"，标题就不准这么写
- **卡片别堆 emoji**：1 张卡最多 1 个 emoji，最好 0 个
- **行业术语翻译成大白话**：「私域转化漏斗」→「让看到的人主动加你」
- **标题字数严格 ≤20**：超 1 个字都不行
- **正文 ≤ 1000 字**：超了用户不读，要么砍要么改发公众号

## 八、输出前自检

- [ ] 3 个标题都从 75 公式来
- [ ] Top 1 推荐 + 一句话理由
- [ ] 正文 ≤ 1000 字，每段 1-3 句
- [ ] 没有"姐妹们"/"家人们"等套话（除非行业本身）
- [ ] 没有"点赞收藏关注三连"
- [ ] 卡片 3-9 张，每张 1 个核心观点
- [ ] tag 5-10 个，大中长尾结构
- [ ] 没编原文没有的数据
