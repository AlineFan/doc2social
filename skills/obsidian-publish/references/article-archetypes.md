# 5 种文章原型

> 来源：精选自 khazix-writer SKILL.md 第四层 + 多素材创作场景调校。
> 用途：用多份素材创作前，先选 1 个原型，按它的结构组织内容。
> 原型选择直接决定：开头方式、章节布局、收尾节奏、读者预期。

---

## 怎么用这个文件

收到一批素材后，问自己：
1. 这批素材**本质上是什么**？（调查 / 体验 / 观察 / 工具 / 方法）
2. **核心读者收获**是什么？（学知识 / 学方法 / 跟你玩一次 / 看你思考）
3. **第一人称姿态**是什么？（实验员 / 体验官 / 观察者 / 分享者 / 老师）

三个问题答案对齐到下面 5 种原型其中之一。**不要硬选**——如果素材不匹配任何原型，告诉用户素材性质混乱，建议拆分或补充。

---

## 一、调查实验型

**核心姿态**：「我替你去做了这件事」

**适用素材**：
- 有"过程记录"（步骤、尝试、失败、突破）
- 有"我亲自下场"的痕迹（第一手数据、试错次数、踩坑细节）
- 有"发现"的时刻

**结构骨架**：
```
钩子：我做了一件听起来不可能 / 反直觉的事
背景：为什么我要做这件事（动机）
过程：第一步...遇到问题...换方法...
发现：层层递进的小发现
结论：最终的大发现 / 反转
升华：这件事让我对 X 有了新理解
```

**关键技法**：
- **过程叙事感**：用时间序"先是...后来...最后..."
- **层层递进的发现**：每个发现比上一个更出人意料
- **亲自下场细节**：具体到日期、地点、数字、对话

**反面**：不要"理论化"——读者要看你做这件事，不要看你讲道理。

---

## 二、产品体验型

**核心姿态**：「跟我一起玩」

**适用素材**：
- 是某个产品 / 工具 / 服务 / 内容的使用记录
- 有"真实使用场景"（具体场景、具体任务、具体感受）
- 与已有方案的对比

**结构骨架**：
```
钩子：我最近在用一个东西 / 发现一个东西
是什么：简单介绍（一两句，不要科普）
真实场景 1：我用它做了 X，感受是 Y
真实场景 2：我用它做了 X'，感受是 Y'
对比：跟以前用的 Z 比，差在哪
卧槽时刻：让我惊讶的一个用法
判断：值不值得用 / 适合谁用
```

**关键技法**：
- **场景演示**：不写功能列表，写"我打开它做了什么"
- **真实感受**：包括不爽的地方，不只夸
- **对比锚点**：跟读者已知的产品做对比

**反面**：不要变成 PR 文——读者一闻到广告味就跑。

---

## 三、现象解读型

**核心姿态**：「你注意到了吗？背后是什么？」

**适用素材**：
- 是对某个社会现象 / 趋势 / 热点的观察
- 有"我注意到的细节"（别人忽略的、自己思考过的）
- 有"追问"的痕迹（为什么会这样？背后是什么？）

**结构骨架**：
```
钩子：最近我注意到 X 现象 / 最近被 X 刷屏
观察：具体细节描述（数据 / 案例 / 场景）
表面解释：通常的解释是 Y，但...
更深追问：实际上可能是 Z
核心洞察：背后的真相
升维：从 X 现象，我想到了更大的 W（文化 / 哲学 / 历史）
```

**关键技法**：
- **观察→好奇→研究→哲学升维**：四段式推进
- **细节先于解释**：先讲你看到了什么，再讲为什么
- **文化升维**：连接到更大的参照物（北京折叠 / 黑暗森林 / 楚门的世界 等）

**反面**：不要纯感慨——读者要的是"独到角度"，不是"我感叹一下"。

---

## 四、工具分享型

**核心姿态**：「我发现了一个好东西」

**适用素材**：
- 是某个工具 / 方法 / 资源的推荐
- 有"个人故事"（为什么我会用上它 / 它解决了我什么问题）
- 有"具体效果"（之前怎样、之后怎样）

**结构骨架**：
```
个人故事铺垫：我之前在为 X 苦恼
工具登场：然后我发现了 Y
怎么用：基础用法（一两个核心场景）
进阶用法：让人惊艳的一个用法
效果对比：之前 vs 之后
适合谁：哪类人会从中受益
```

**关键技法**：
- **个人故事铺垫**：不要直接 sell，先讲你的痛
- **效果惊艳**：必须有一个让人"卧槽"的演示
- **适合人群明确**：不要"适合所有人"，要"适合 X 这样的人"

**反面**：不要变成功能罗列——读者要的是"你用过这个东西的真实感受"，不是说明书。

---

## 五、方法论分享型

**核心姿态**：「我把压箱底的掏给你了」

**适用素材**：
- 是某个领域的经验总结（多次实践后的提炼）
- 有"我的具体步骤"（不是空泛的"建议"）
- 有"我也踩过的坑"（坦诚说失败点）

**结构骨架**：
```
谦逊开头：我也不知道行不行 / 这套不一定对 / 我自己也还在摸索
为什么这个方法重要：背景动机
方法 1：具体步骤 + 我的例子
方法 2：具体步骤 + 我的例子
方法 3：具体步骤 + 我的例子
失败点 / 学习成本：诚实说哪里难、哪里容易踩坑
谦逊收尾：不一定适合所有人，仅供参考
```

**关键技法**：
- **可执行**：每一条读完后，读者手里有一个"今天就能执行的动作"
- **坦诚学习成本**：明确说"这套需要 X 天 / 需要先学 Y"
- **谦逊铺垫**：开头和结尾都用谦逊语气，避免"老师味"
- **真实失败点**：必须有，没失败点的方法论是不可信的

**反面**：不要装权威——你不是李笑来 / 罗永浩，你是分享心得的普通人。

## 原型选择决策树

```
素材里有"我做了 X，记录了过程" → 调查实验型
素材里有"我在用 X，感受是 Y"  → 产品体验型
素材里有"我观察到 X 现象"     → 现象解读型
素材里有"我发现了 X 工具"     → 工具分享型
素材里有"我总结了 N 个方法"   → 方法论分享型

素材不属于任何一种：
→ 告诉用户："这批素材性质偏 X，但缺乏 Y，建议补充。要不要继续往 Z 原型靠？"
```

## 原型适配三平台

| 原型 | 公众号 | X 线程 | 小红书 |
|------|--------|--------|--------|
| 调查实验型 | ✅ 极佳，长文叙事 | ✅ 好，每条是一个发现 | ⚠️ 难，要简化成"我做了 X，结果是 Y" |
| 产品体验型 | ✅ 好 | ✅ 好，每条是一个场景 | ✅ 极佳，每张卡是一个场景 |
| 现象解读型 | ✅ 极佳 | ✅ 好，每条是一层追问 | ⚠️ 难，升维段在小红书容易掉 |
| 工具分享型 | ✅ 好 | ✅ 极佳，hook + 进阶用法 | ✅ 好，每张卡是一个用法 |
| 方法论分享型 | ✅ 好 | ⚠️ 中等，结构感强但拆条难 | ✅ 极佳，每张卡是一个方法 |

⚠️ 标记的组合：依然出，但**告诉用户哪个平台是弱项**，建议优先发哪两个。

---

## Gotchas

- **不要混合原型**：一篇文章只用一个原型。混合会让读者跟丢节奏
- **原型不是模板填空**：原型是骨架，每个素材填进去后要再揉一遍，让它读起来不是"按格式来"
- **现象解读型 + 文化升维**：升维段必须**自然连接**（"聊着聊着想到..."），不要硬升（"这让我想到了存在主义"）
- **方法论分享型 + 失败点**：失败点不写就是不可信，但**不要长篇大论说自己多惨**
- **5 种以外的原型**：日记型、故事型、综述型、新闻型——这些**不在我们 skill 范围**，遇到告诉用户走人工创作
