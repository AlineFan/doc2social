PART




灵感 · 从抖音到游戏

INSPIRATION

刷抖音的时候看到一类很火的「错位时空」创意剪辑——把不同时代的画面叠在一起，让两个时空里的人隔着时间对望。

当时就在想：如果这个概念能交互呢？不只是看，而是让玩家自己去拖动时间窗、拼线索、替一个回不来的人把话送到

这就是《错位时空：送信者》的起点。先说游戏剧情。在民国小镇，一个母亲每天傍晚坐在路口等参军的儿子回家吃饭。儿子阿岩其实已经死在战场上了，临死前手里还攥着母亲系给他的红绳，留了一封没写完的信。你作为送信人，要跨越几个时代，收集线索，把这封信用她能接受的方式递到母亲手里。

你不能复活任何人，不能改变历史

让一段苦苦的等待结束

02

PART




玩法 · 时间窗+AI NPC

GAMEPLAY

这个游戏有两层玩法叠在一起。

第一层：时间窗

屏幕上会陆续显影几扇时间窗。每扇窗都是一个时代切片：古代田埂石墩、民国战时军邮点、民国公园木椅。

以现代海边作为底图，过去景象像旧照片一样浮在画面里，你拖动时间窗、点窗里的人物，慢慢把散落在不同年代的碎片拼起来。

还有一个我特别设计的机制叫"时窗共振"：把两扇相关的时间窗拖近，画面会出现共振提示。

比如把儿子阿岩的时间窗拖到母亲所在的时间窗口旁边，触发"母子共振"，系统会给你一条对话提示，告诉你下一步可以跟 NPC 聊什么方向

第二层：NPC 对话

游戏里每个能对话的 NPC，都是一个独立的 AI 智能体。没有预设回复台词，接了DeepSeek V4模型，每个 NPC 有自己的记忆、情绪和判断。

独立记忆

你跟古代的老周聊过"送信人规矩"，他会记住。下次再找他，不会重复讲。

信任度

NPC 对你有戒备心。刚见面问敏感问题，她不说。得先聊别的建立信任，她才愿意透露关键线索。

渐进披露

NPC 不会一上来把所有事都说完。封闭 → 试探 → 动摇 → 接受，每一层都跟你说过什么、信任够不够、有没有抓到关键物件有关。问了不一定答，做对了才开口。

两层叠起来就是解谜流程。时间窗给你场景，对话给你信任，残片给你证据，缺一不可。

最终，你收集到足够的信物、时机和措辞，自己写一封母亲"能接受"的信：

递信成功后，母亲不再只望路口。她转身点灯，坐回屋里。

这种治愈虽然不是大团圆，但能让苦苦等待的人有一个放下的理由。

团队 · 1人+7个AI

TEAM

好，重点来了。

这么多系统、这么多素材、这么多文案，背后只有我1 个人是怎么搞定的？

用的是CodeBanana的在线多智能体协作平台上开展的这个项目。简单说，它就像一个飞书工作群，但你的同事可以是 AI Agent。每一个智能体有自己的和能力和工作空间，你可以在项目群里 @ 它就跟 @ 同事一样。

我的团队：

游戏 UI 和交互

游戏测试

自动化测试，找 bug

所有人（包括 AI）在同一个群聊里协作，共享同一个代码仓库，通过 Git 同步项目。A 改完代码提交，B 立刻能看到。

04

PART




过程 · 全流程实战

PROCESS

STEP 01
聊出世界观

在群聊里 @ 叙事设计师阿纸，告诉它"我想做一个关于送信和时间的治愈游戏，灵感是抖音上的错位时空"。

阿纸给了好几版方向，我选了"不改命、只递信"这个，它接着就把完整的三幕结构、NPC 人设和信件文案全搭出来了。

STEP 02
搭 NPC 对话系统

@ AI 对话工程师，让它基于DeepSeek V4搭 NPC 的记忆和情绪系统。

STEP 03
做游戏 UI

@ 前端工程师，让它写时间窗拖拽、NPC 对话面板、邮差包 UI、递信仪式界面。

STEP 04
出美术素材

@ 视觉设计师，让它用Image-2批量出场景和角色。

海边黄昏长椅、民国公园、古代村落，这些背景全是视觉设计师智能体调用 Image-2 skill批量生成的。因为有完整上下文，所以说画风匹配度都很统一。

STEP 05
测试

@ 游戏测试，让它跑自动化测试。改完代码它自动跑一遍，红了直接告诉我哪挂了。

STEP 06
做宣传视频

这步比较特别，我给宣传智能体配了两个 SKILL 技能：

HyperFrames Skill

最近很火，用 HTML 做视频动画确实比传统剪辑灵活多了。

Seedance2 Skill

负责生成相关提示词，调用api生成视频

两个技能叠加，宣传智能体自己就把80 秒宣传 MV 的素材链路跑通了。

效率 · 为何这么快

EFFICIENCY

两天跑完全链路，说说我感受最明显的几个点。

快，不只是多个 Agent同时进行工作。

真正关键的是：大家在同一个项目空间里，讨论、代码、文件、上下文都不乱。

Team Agent 负责真正动手写代码；Private Agent 可以用来提前调研、验证方案；Discussion 里把想法聊清楚，再转给 Agent 执行。

云端跑代码不占我电脑资源，还有Claude Code以及Opus 4.7等最新模型，智力天花板够高，返工少了，效率自然上去了。

这个项目有不少工作是我在路上推进的，手机打开CodeBanana APP@智能体，就能让它继续干活。还可以远程盯进度、审结果、调方向，像遥控器一样远程指挥作战。

磨合几天之后效率有明显提升，因为 Agent 有记忆能力和我不断优化调整skill，它们会越来越懂项目、越来越少跑偏。
