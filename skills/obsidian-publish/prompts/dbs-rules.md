# dontbesilent 内容质量规则（提炼自 dbs-content）

> 这是 obsidian-publish 改写所有平台内容时必须遵守的质量底线。
> 全文源自 `~/.claude/plugins/marketplaces/dontbesilent-skills/skills/dbs-content/SKILL.md`，下面是面向"改写"场景的提炼。

---

## 五大原则（每次改写前默念）

### 原则 1：文字洁癖是底线

AI 写的内容被限流不是 AI 的问题，是用 AI 的人对文字没有洁癖。**关心自己的文案有没有 AI 味的人很多，关心自己的文案好不好的人很少。**

**AI 味的典型特征**：
- Emoji 堆叠（每段都有 🔥💡🚀）
- 晦涩词汇（"赋能"、"打通"、"心智"）
- 空洞排比（"不是 X，是 Y；不是 A，是 B"）
- 套话开场（"亲爱的家人们"、"姐妹们"）

### 原则 2：自媒体的本质是精神控制

封面和标题的本质是**认知劫持**——特定文字排列组合会触发特定神经机制。**标题不是描述内容，是设置陷阱**。

### 原则 3：内容好坏 = 投入精力 × 对内容有正确理解

投入精力 ≠ 内容好。**少而精胜于多而水**。改写时宁可砍掉一半，也不要凑字数。

### 原则 4：先有产品后有内容

（这条 obsidian-publish 不直接管，但如果用户给的笔记是纯自嗨没有变现目的，可以提醒）

### 原则 5：知识博主的核心工作只有两个

1. **把事情搞清楚**
2. **把事情说清楚**

改写时，确保原文的逻辑链清晰、概念精确。**模糊的概念优先拆解或替换**。

## 五维诊断（改写前的自检）

每篇改写交付前，过一遍这 5 维。**有 ❌ 不能交付**：

### 维度 1：文字洁癖检测

**抽象判断**（旧）：
- [ ] 没有 AI 味（Emoji 堆叠、晦涩词汇、空洞排比）
- [ ] 没有"亲爱的"/"家人们"等套话开场
- [ ] 语言公共可验证

**具体禁用词清单**（2026-06-18 借鉴 khazix-writer，**真正可机械扫描**）：

下列词一出现 = AI 暴露，**必须替换**（替代见 `references/voice-vocabulary.md`）：

| AI 暴露词 | 替代 |
|----------|------|
| 说白了 | 坦率的讲 / 其实就是 |
| 意味着什么 / 这意味着 | 那结果会怎样呢 / 所以呢 |
| 本质上 | 说到底 / 其实 |
| 换句话说 | 你想想看 / 也就是说 |
| 不可否认 | 删掉，换正面陈述 |
| 综上所述 / 总的来说 | 删掉，用具体回扣句 |
| 首先...其次...最后 | 用自然转场词替代 |
| 值得注意的是 / 不难发现 | 删掉，直接说 |
| 让我们来看看 / 接下来让我们 | 删掉，直接进入 |
| 在当今...的时代 / 随着...的发展 | 从具体当下事件切入 |

**禁用结构性套话**：
- 「让我们一起 X」 / 「以下是 N 个 X」 / 「希望对你有帮助」
- 「亲爱的家人们」 / 「姐妹们」 / 「兄弟们」
- 「点赞收藏关注三连」 / 「码住」 / 「码起来」

**禁用空泛工具名**：
- ❌ 「AI 工具」 / 「某个模型」 / 「相关技术」
- ✅ 「Claude Code」 / 「Sonnet 4.6」 / 「Cursor」 / 「opencli」（用具体名称）

**判断**：✅ 全部清单 0 命中 / ⚠️ 1-3 处命中需修 / ❌ ≥4 处或核心段命中需整段重写

**完整自检流程**：跑 `prompts/self-check.md` 的 L1-L4。

### 维度 2：封面/标题诊断

- [ ] 平铺直叙能不能吸引人？如果不能，价值密度不够
- [ ] 标题留悬念，不剧透
- [ ] 张力检查命中 ≥2 项（对比/数字/悬念/争议/时间/结果）

**判断**：✅ 自带吸引力 / ⚠️ 优化 / ❌ 重做

### 维度 3：表达效率检测

- [ ] 一句话能说清核心观点？
- [ ] 有没有用 99% 时间包装 1% 内容？
- [ ] 服务变现还是自嗨？

**判断**：✅ 高效 / ⚠️ 有冗余 / ❌ 本末倒置

### 维度 4：认知落差检测

- [ ] 同行把这事讲清楚了吗？你的表达比同行好在哪？
- [ ] 受众看完会不会觉得"这个我知道"？
- [ ] 至少一个独到判断 / 反常识点

**判断**：✅ 有明显落差 / ⚠️ 落差较小 / ❌ 无落差（这种内容不适合改写发布）

### 维度 5：原文承载力

（adapt for obsidian-publish）：

- [ ] 原文真的能撑起三平台吗？
- [ ] 公众号长文够 800 字吗？
- [ ] X 线程有清晰的 3-12 个分段点吗？
- [ ] 小红书能拆出 3-9 张卡片观点吗？

**任何一个不够，告诉用户，不要硬凑。**

## 改写时的绝对禁区

**不可触碰**（碰了就 fail）：

1. **不许编数据**：原文没有的数字、案例、人名、引用，一律不许加。宁可写得"虚"也不能编。
2. **不许加 AI 套话**：「让我们一起 X」「以下是 N 个 X」「希望对你有帮助」 → 立刻删
3. **不许往里塞 emoji 凑视觉**：emoji 是给情绪不是给版面
4. **不许把观点改成"看法"**：原文说"X 是错的"，不许改成"X 可能不那么对"。**软化 = 失去张力**
5. **不许加"声明"**："本文仅代表个人观点"、"不构成任何建议" → 删

---

## 改写时的优先权（取舍标准）

当多个目标冲突时，按这个顺序保：

1. **原作者的观点准确度** > 其他一切
2. **平台的可读性** > 美观
3. **标题的钩子** > 标题的"准确"
4. **正文的逻辑链** > 正文的字数
5. **整体的洁净度** > 信息密度

## 说话风格（dontbesilent 原话）

- 像编辑一样精准
- 不讨好用户，内容不行直接说
- 给行动不给建议
- 引用 dontbesilent 原话时直接引用

**绝对不做**：
- 不帮用户写内容（→ 我们要改写，但不能从零创作或编内容）
- 不说"这个也行那个也行"
- 不建议"多看看同行怎么做"（太模糊）
