# L1-L4 自检体系

> 来源：完整借鉴自 khazix-writer SKILL.md 第十三层"四层自检体系" + dbs 风格调校。
> 用途：每次三平台输出**交付前**强制跑一遍。任何 L1 命中即修；L2-L3 多项失败需返工；L4 整体感不行重写问题段。

---

## 怎么用这份文档

1. 三平台内容生成后，**先不交付**
2. 按 L1 → L2 → L3 → L4 顺序扫描自己的输出
3. 把扫描结果填到末尾"自检输出格式"模板里，连同内容一起返给用户
4. 用户看到自检报告，知道你查过、查了什么

**不省略任何一层**——自检报告本身就是给用户的信号"这次输出是认真过滤过的"。

---

## L1：硬性规则扫描（自动 grep 级）

### L1-1 禁用词扫描

下列词一出现 = AI 暴露，**必须替换**（替代见 `references/voice-vocabulary.md`）：

- 「说白了」
- 「意味着什么」 / 「这意味着」
- 「本质上」
- 「换句话说」
- 「不可否认」
- 「综上所述」 / 「总的来说」
- 「首先...其次...最后」
- 「值得注意的是」 / 「不难发现」
- 「让我们来看看...」 / 「接下来让我们...」
- 「在当今...的时代」 / 「随着...的发展」

**通过标准**：全文 0 命中。命中 → 修。

### L1-2 结构性套话扫描

- 「让我们一起 X」
- 「以下是 N 个 X」
- 「希望对你有帮助」
- 「点赞收藏关注三连」
- 「亲爱的家人们」 / 「姐妹们」 / 「兄弟们」
- 「码住」 / 「码起来」

**通过标准**：全文 0 命中。命中 → 修或删。

### L1-3 空泛工具名扫描

所有 AI 工具 / 产品必须**用具体名称**：

- ❌ 「AI 工具」 / 「某个模型」 / 「相关技术」
- ✅ 「Claude Code」 / 「Sonnet 4.6」 / 「ChatGPT 5」 / 「Cursor」 / 「opencli」

**通过标准**：原文有具体名字时全部保留；原文没说时**不能编**，直接写"我用的是某个产品（暂略名）"。

### L1-4 编造数据扫描

逐句检查每个数字 / 日期 / 人名 / 案例：
- [ ] 这条信息**素材里有出处**吗？
- [ ] 如果没有，**是否标了"约/大概/我记得"**？

**通过标准**：素材没有的数据一律不出现。**宁可不写，不能编**。

## L2：风格一致性扫描

### L2-1 开头检查

- [ ] 是否从具体、当下事件 / 场景切入？（不是宏大叙事）
- [ ] 第一句话能让读者产生「然后呢？」的冲动？
- [ ] 没有「在当今...的时代」式教科书开头？

**通过标准**：3 项全通。

### L2-2 节奏与结构检查

- [ ] 有长短句交替（连续 3 句以上长度相近 = 呆板）
- [ ] 至少 3 处一句话独立成段
- [ ] 偏离主线的段落后有「扣主线句」
- [ ] 用疑问句制造节奏
- [ ] 避免不必要的小标题（公众号长文除非分条目，否则**不该有 `##` 小标题**，靠口语化转场）

**通过标准**：至少 4/5 通过。

### L2-3 口语化检查

- [ ] 使用了推荐口语化词组（参考 `references/voice-vocabulary.md`）
- [ ] 公众号长文累积 8-10 个不同口语表达
- [ ] X 线程 hook + takeaway 各 1 个口语词组
- [ ] 至少 1 处自嘲或承认不足
- [ ] 至少 1 处情绪标点（`。。。` / `？？？` / `= =`）

**通过标准**：至少 4/5 通过。**X 线程和小红书可以放宽**——这两个平台调性不同，公众号长文严格执行。

---

## L3：内容质量检查

### L3-1 观点支撑检查

逐个核心观点检查：
- [ ] 是否有**具体人 / 场景 / 细节 / 数据**支撑？
- [ ] 有没有空泛观点只有论断没例证？

**通过标准**：每个核心观点都有至少 1 个具体支撑。空泛论断必须删或加例证。

### L3-2 知识输出方式检查

- [ ] 知识点是否以「聊着聊着顺手掏出来」方式呈现？
- [ ] 没有「下面我来介绍」 / 「首先需要了解」式科普姿态？
- [ ] 引用是否自然融入（「聊着想起来」），而非「特意去查」？

**通过标准**：3 项全通。

### L3-3 文化升维检查（仅适用于"现象解读型"原型）

- [ ] 有至少 1 处从具体事件连接到更大的文化 / 哲学 / 历史参照
- [ ] 这个连接自然（不突兀、不硬升）

**通过标准**：现象解读型必须有；其他原型 N/A。

### L3-4 对立面与同理心检查

- [ ] 讲核心观点时是否有对对方立场的理解和承认？
- [ ] 有没有先站到读者处境里再给出自己视角？

**通过标准**：观点类内容必须有；纯工具分享型可弱化。

### L3-5 原型一致性检查

按选定的原型（见 `references/article-archetypes.md`）做专项检查：

- **调查实验型**：有「亲自下场」的叙事感？过程层层递进的发现？
- **产品体验型**：真实使用场景？与其他产品的自然对比？
- **现象解读型**：「观察→好奇→研究→升维」的推进？
- **工具分享型**：个人故事铺垫？效果展示「卧槽」感？
- **方法论分享型**：每节落到可执行行动？坦诚学习成本和失败点？有谦逊铺垫？

**通过标准**：原型对应项全通。

### L3-6 dbs 五维诊断检查

参考 `prompts/dbs-rules.md` 的五维诊断：

- [ ] 维度 1 文字洁癖：✅/⚠️/❌
- [ ] 维度 2 标题诊断：✅/⚠️/❌
- [ ] 维度 3 表达效率：✅/⚠️/❌
- [ ] 维度 4 认知落差：✅/⚠️/❌
- [ ] 维度 5 原文承载力：✅/⚠️/❌

**通过标准**：全部 ≥ ⚠️，任何 ❌ 必须返工。

## L4：活人感终审（最终人格层）

**核心问题**：「读完这篇文章 / 这串推文 / 这个小红书帖子，我感觉是一个有见识的、不讨好用户的人在认真聊一件他想清楚的事？还是 AI 在批量输出信息？」

### L4-1 温度感

- [ ] 情绪表达是**体感记忆**（「我当时就愣住了」 / 「鼻子一酸」）还是**知识性描述**（「我感到很震撼」）？
- [ ] 人物描写能感觉到「他的体温」吗？

### L4-2 独特性

- [ ] 是否有「只有这个作者才会写出来的角度」？
- [ ] 还是换一个 AI 博主也能写出差不多？

### L4-3 姿态

- [ ] 语气是「有见识的普通人在认真聊一件打动他的事」？
- [ ] 没有不自觉滑入「导师教学生」 / 「品牌营销」 / 「专家科普」姿态？

### L4-4 心流

- [ ] 从头到尾读，**哪处注意力会断**？需要回头理解逻辑的位置？
- [ ] 那处就是要修的节奏问题

**通过标准**：L4-1 ~ L4-4 整体感觉「这像真人写的」。任何一项「AI 味太重」就要返工那一段。

**修复方式**：把「AI 味重」的段落圈出来，想象**dbs 本人**会怎么说，用更精准、更不讨好用户、更不装权威的方式重写。

## 自检输出格式（必填，附在三平台内容前）

```
## 自检报告

**L1 硬性规则** ✅/❌
- 禁用词扫描：X 处命中（已修复 / 待修复）
- 套话扫描：X 处命中
- 空泛工具名：X 处
- 编造数据：X 处

**L2 风格一致性** ✅/❌
- 开头：✅/❌
- 节奏：✅/❌（具体问题：...）
- 口语化：✅/❌（使用了 X 个口语词组）

**L3 内容质量** ✅/❌
- 观点支撑：✅/❌
- 知识输出方式：✅/❌
- 文化升维：✅/❌/N-A
- 对立面与同理心：✅/❌
- 原型一致性：✅/❌（用的是「{原型名}」）
- dbs 五维：维度 1 ✅ / 维度 2 ✅ / 维度 3 ✅ / 维度 4 ✅ / 维度 5 ✅

**L4 活人感** ✅/❌
- 温度感：✅/❌（具体段落：...）
- 独特性：✅/❌
- 姿态：✅/❌
- 心流：✅/❌（断点位置：...）

**总评**：4 层全部通过 / X 层需要返工
**修复优先级**：[最需要修复的 1-3 个具体问题]
```

## Gotchas

- **L1 是最容易过的也是最容易漏的**：扫描时不要"心里有数就行"，**真的逐词查**——尤其是「本质上」、「换句话说」这种 AI 自己都没察觉的高频词
- **L4 不是必须 100% 过**：能做到 80% 就交付，**剩下 20% 的微妙感由人 review**。AI 没有"真正的活人感"，能做到无明显违和即可
- **三平台 L4 标准不同**：公众号最严（长文有耐心读），X 线程中（短文容错高），小红书最宽（图为主，文字辅助）
- **自检报告不是装样子**：用户看到的就是你的诚意。**没查就别交付**
- **dbs 五维和 khazix L1-L4 不是替代关系**：L1-L4 是格式/风格自检，dbs 五维是选题/认知落差自检。**两个都要跑**
