# note-to-xhs：笔记 → 小红书图文物料

把一篇 markdown 笔记，一条龙做成小红书可发的物料：**可选卡片样式的 3:4 图片 + 标题 + 正文文案 + 话题标签**。

## 输入

- **必填**：1 个 markdown 笔记路径（通常在 Obsidian vault）
- **可选**：模板、署名、日期、头像；`X长文` 可额外提供置顶 banner。
- **默认模板**：`X长文`。用户未指定时不要擅自换成其他视觉风格。

## 产出（4 件）

1. 1 张完整 3:4 单卡，或多张连续 3:4 PNG（按内容长度自动决定）
2. 小红书标题（dbs 公式匹配，多候选 + Top 3）
3. 小红书正文文案（按固定内容风格，≤100 字）
4. 话题标签

## 流程（4 步，按顺序执行）

### Step 1 · 用 doc2social 渲染卡片

当前唯一的模板真源是 `/Users/doushun/Desktop/workspace/proj04-obsidian-publisher/web/index.html`，线上入口为 `https://alinefan.github.io/doc-to-social-render/`。使用网页的小红书页：连接/拖入 Obsidian 文件夹，选择笔记，选择样式，确认预览后导出 PNG。

- 内置模板：`极简风`、`X长文`、`莫兰迪`、`彩色卡`、`茶色笺`、`原木笺`、`点阵纸`。
- `X长文`：社交长文卡，支持头像、handle 和 banner；适合信息密度高的内容。
- `极简风`：干净的白底阅读卡；`莫兰迪`：彩色便签头；`彩色卡`：亮黄、亮紫或亮薄荷绿承托的小票，**只用纯色，不用渐变**；`茶色笺`、`原木笺`、`点阵纸`：纸张/信笺方向。
- 标题优先级：笔记的 H1，其次 frontmatter 的 `title`，最后才使用文件名。标题只用于卡片头部，不得在正文中重复一遍。
- 默认关闭“单卡 3:4”。用户明确要短文一图展示时才开启；只有内容确实放得下时才导出单卡，否则保持多张自动切页，**不要压缩字体或裁切正文**。
- 每张导出图固定为 1196×1594（3:4）。`彩色卡`、`茶色笺`、`原木笺` 的长文分页会为**每张** PNG 重建完整的上下左右外层背景和内部纸面，不把一张长纸直接横切；文字在安全行切点分页，图片可跨页但不丢失内容。检查首张标题、末张 footer 和图片是否完整；发现图片找不到时，明确告诉用户文件名。

旧脚本 `scripts/note-to-xshots.sh` 只保留作网页不可用时的 X 长文兜底，不能用于宣称支持上面的其他六种模板。

### Step 2 · 生成标题（先用 dbs-xhs-title）

调用 `/dbs-xhs-title` skill，把笔记核心内容 + 已定稿的正文文案一起传给它（标题要和文案咬合）：

- 让它按 75 个爆款公式匹配，输出 5–8 个候选（标注公式编号）+ Top 3
- 标题硬规范：≤20 字（含标点）、留悬念不剧透、用普世词扩大流量池
- Top 1 通常用作首图大字封面

### Step 3 · 生成正文文案（用固定内容风格）

**先读 `references/caption-style.md`**，严格照那套风格写正文：

- ≤100 字、开头直接抛反差观点、对仗金句收尾、0 emoji、不强行加互动钩子
- 写完过一遍 caption-style.md 末尾的自检清单
- 如果用户已经给了定稿文案，**直接用用户的，不要自作主张改写**

### Step 4 · 配话题标签

- **核心固定标签**（AI/成长赛道）：`#AI #人工智能 #ai工具 #认知升级 #独立思考 #深度思考 #个人成长 #普通人如何用ai #aigc #自我提升`
- 按这篇的具体主题增删 2–3 个更精准的（如讲 Claude Code 就加 `#claude #AI编程`）
- 总数控制在 8–12 个

## 最终输出格式

给用户一张「发布物料卡」，四样齐全、可直接复制：

```
📂 图片：<导出目录>（<模板名>，N 张 3:4；或 1 张单卡）

【标题】Top 1（+附 2 个备选）
<标题>

【正文】
<≤100 字文案>

【标签】
<8–12 个 # 标签>
```

末尾提示：标题想换 Top 2/3、文案想调长短，直接说。

---

## 边界

- ❌ 不自动发布到小红书（v1 手动发；未来可接 `opencli xiaohongshu publish`）
- ❌ 不改写正文长文内容（那是 obsidian-publish 的活；本 skill 只做「成品笔记 → 小红书物料」）
- ❌ 一次只处理一篇 md
- ❌ 不把旧 `note-to-xshots.sh` 的 X 长文效果冒充为其他模板；模板能力以 doc2social 网页为准
- 文案风格以 `references/caption-style.md` 为准；标题方法以 dbs-xhs-title 为准
