图片化设计：开头是设计好的整版 PNG 大标题（『01 风油精 年轻人爱疯了』焊在像素里），正文以浅绿描边（rgb(168,229,138)）大配图为主、文字极简——15px 黑字 + 2px 字距，墨绿强调，居中灰色图注。可克隆的 CSS 很薄，视觉门面在图里。

⚠ 本风格含 3 个图片组件（hero_image、image_block、footer_gif）——图片不是 CSS 排版、克隆不了，套你自己内容时需换图/重做。其余 7 个是可克隆的 CSS 组件。

根容器：正文区最外层基础排版。文章里 99% 的文字字号/字体/行高/字距都从这里继承——正文 body_text 的 span 才显式重复这串，但根容器是真正的样式源头（事实取自占比最高的正文 span ×132）。块级 wrapper 只负责左右 16px 留白。

正文段落。块级 wrapper 是 <p style='margin:0px 16px'>（同款也出现为 <section style='margin:0px 16px'>），左右各 16px 边距、无其他装饰；内部文字 span 重复根容器那串排版并额外加 letter-spacing:2px（中文加宽字距是本风格的标志）。套用时段落文字包 inner_run 的 style、外层 <p> 包 block 的 style。

分节/收尾用的加粗行内标签——本文唯一具备『标题视觉角色』的元素（文章没有任何大字号 h1/h2，h2 标签是空 <br> 占位、非真标题；真正的大标题被做成了开头那张 hero_image PNG）。形态是正文段里一段 font-weight:bold 的 inline run（如『ENDING：』『【今日话题】』），靠加粗 + 多半居中独占一行来分节，字号仍是 15px。套用时：只有用户内容确有『分节标签 / 小标题』语义才用；纯粹是把一行字加粗，序号/步骤语义不要硬塞。

行内示例：【今日话题】 嵌在正文里。

墨绿色高亮强调：正文里把金句/观点片段染成深绿（rgb(64,118,0)）的 inline run，全文出现 18 次，是本风格唯一的强调色（呼应配图的浅绿边框）。仍带 2px 字距、字号继承。套用时包在用户想强调的句子上。

行内示例：最通透的年轻人，把风油精盘成了时尚单品。 嵌在正文里。

★ 开头图片化大标题：整版设计好的 PNG，标题文字『01 风油精 年轻人爱疯了』焊死在像素里。位于正文最顶端、居中、无描边（区别于下方正文配图的浅绿边框）——这是整篇的视觉门面。单独登记为 hero 组件，不与正文配图混。CSS 能复刻的只有『居中 section + margin:0 16px』这一薄层，标题设计本身在图里。原图真实地址已抓进 SRC 槽位。

★ 正文配图块：<section> 居中包一张带浅绿实线边框（rgb(168,229,138) 1px solid + 透明底）的图。三种尺寸形态——①满宽图 width:100%；②定高图带固定 width/height（如 546x832、546x748、544x768 等）；③原生尺寸图无宽高。共 22 张正文配图，每张真实 src 都已抓在 SRC 槽位的 sources 列表里。

图注 / 图源说明：配图下方一行，灰色（rgb(136,136,136)）14px（比正文小一号）的 inline run，文字形如『图源：xxx』。块级 wrapper 同 body_text 用 <p style='margin:0px 16px'>（偶尔居中）。套用时只有用户给了图片来源/说明才出现，参考的署名（图源：银河海盗舰长 等）不进产出。

文末作者名片：<section> 内嵌 <mp-common-profile> 公众号自动卡片（空自定义元素，内容由公众号后台自动渲染，原文里无任何可提取的内联样式）。非排版资产、不承载用户文字——套用时一般直接省略，留它仅为忠实记录原文结构。

文末互动区：居中『【今日话题】』加粗标签 + 一句话题问句（沿用 heading_label 的加粗 inline run + 居中块）。【今日话题】是与内容无关的纯版式标签，可作常量直出；问句必须换成用户自己内容。
